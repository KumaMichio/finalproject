1. Định nghĩa bài toán
Input
Nhiều camera CCTV đặt trong môi trường CARLA
Stream video đồng thời (multi-view)
Output
Theo dõi object (vehicle, pedestrian) xuyên camera
Gán ID toàn cục (global identity)
Dự đoán quỹ đạo di chuyển trong tương lai
Cảnh báo cho các camera hoặc khu vực phía trước
2. Kiến trúc hệ thống (logic)

Hệ thống gồm 5 lớp chính:

1. Data Layer
CARLA simulation (map, actor, camera)
Multi-camera synchronized stream
2. Perception Layer (per-camera)
Object Detection
Multi-object Tracking
3. Cross-camera Fusion Layer
Re-identification (ReID)
Global ID assignment
4. Behavior Modeling Layer
Trajectory prediction
Motion modeling
5. Application Layer
Alert system
Visualization / monitoring
3. Các module chính cần xây
3.1 Simulation & Data Acquisition
Mục tiêu
Sinh dữ liệu đa camera có kiểm soát
Nội dung
Đặt nhiều camera tại các tuyến đường
Gán metadata:
camera ID
vị trí (world coordinate)
hướng nhìn (orientation)
Đồng bộ frame (synchronous mode)
Output
Multi-stream video + timestamp
3.2 Object Detection
Mục tiêu
Phát hiện object trong từng frame
Model đề xuất
YOLOv5 / YOLOv8
Faster R-CNN (nếu cần chính xác cao hơn)
Class quan tâm
car
truck
bus
pedestrian
Output
Bounding box + confidence + class
3.3 Multi-Object Tracking (Single Camera)
Mục tiêu
Gán ID tạm thời trong từng camera
Model / thuật toán
ByteTrack (khuyến nghị)
DeepSORT (dễ implement hơn)
Output
Track ID (local)
Trajectory trong camera
3.4 Cross-Camera Re-Identification (ReID)

Đây là phần quan trọng nhất.

Mục tiêu
Nhận diện lại object khi chuyển giữa camera
Input
Crop ảnh object từ tracking
Feature embedding
Model đề xuất
OSNet (nhẹ, hiệu quả)
ResNet50 ReID
FastReID (framework mạnh)
Output
Vector đặc trưng (embedding)
Matching
Cosine similarity
Threshold-based matching
3.5 Global Tracking
Mục tiêu
Hợp nhất ID từ nhiều camera thành 1 ID toàn cục
Logic
Khi object xuất hiện:
So sánh embedding với database
Nếu match → gán Global ID
Nếu không → tạo ID mới
Output
Global ID
Lịch sử di chuyển đa camera
3.6 Trajectory Prediction
Mục tiêu
Dự đoán vị trí tương lai của object
Input
Chuỗi vị trí theo thời gian
(Optional) map topology
Model cơ bản
Linear motion (baseline)
Model nâng cao
LSTM / GRU
Social LSTM (có tương tác giữa object)
Transformer-based trajectory model
Model nâng cao hơn (khuyến nghị nếu có thời gian)
Map-aware prediction
Graph-based trajectory prediction
Output
Future positions (x, y)
Hoặc distribution (multiple possible paths)
3.7 Alert System
Mục tiêu
Cảnh báo khi object di chuyển đến khu vực khác
Logic
So sánh trajectory dự đoán với:
vị trí camera khác
vùng quan tâm (ROI)
Output
Danh sách camera bị ảnh hưởng
Thời gian dự đoán đến
3.8 Visualization / Interface
Mục tiêu
Hiển thị hệ thống cho người dùng
Nội dung
Bounding box + ID
Trajectory (past + predicted)
Camera switching
Alert notification

Lưu trajectory
